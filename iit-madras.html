IIT Madras Homepage Clone

A responsive and pixel-perfect recreation of the official IIT Madras website homepage, focusing on foundational front-end development skills.

Live Demo View Source Code

Technologies Used

  • HTML5: For structuring the content and semantics.
  • CSS3: For styling, layout (Flexbox/Grid), and responsiveness.

Project Overview

The primary goal of this project was to meticulously replicate a complex, real-world website to solidify and showcase core front-end skills. The IIT Madras homepage was chosen for its intricate layout, multi-level navigation, and diverse content sections, presenting a significant challenge in pure HTML and CSS. The project demonstrates a strong command of semantic HTML structure and advanced CSS techniques for layout and responsive design.

Key Features

  • Pixel-Perfect Replication: The design closely mirrors the original site's layout, spacing, typography, and color scheme.
  • Fully Responsive Design: The page is fully functional and aesthetically pleasing across all device sizes, from large desktops to small mobile screens.
  • Complex Navigation Menu: Recreated the multi-level dropdown navigation menu, ensuring it is fully responsive and collapses into a mobile-friendly hamburger menu.
  • CSS Grid & Flexbox Layout: Utilized modern CSS layout techniques like Grid for the main page structure and Flexbox for component alignment, ensuring a robust and maintainable structure.

Challenges & Problem-Solving

The Challenge:
One of the main difficulties was ensuring the complex grid of news, events, and announcement cards would reflow elegantly on smaller screens without breaking the layout or becoming unreadable. The original site uses a multi-column layout that is challenging to adapt for a single-column mobile view.

The Solution:
I leveraged CSS Grid with the grid-template-areas property for the desktop layout, which provided a clear and visual way to structure the page. For responsiveness, I used CSS Media Queries to redefine the grid layout at different breakpoints. On mobile screens, the grid was reconfigured to a single column, and the order property in Flexbox was used within certain parent containers to rearrange key elements for a better mobile user experience, such as moving a call-to-action button to a more prominent position.
