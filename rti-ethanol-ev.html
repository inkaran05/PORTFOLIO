RTI Ethanol EV Transparency

An interactive dashboard providing real-time transparency into India's green energy initiatives using React and public APIs.

Live Demo View Source Code

Technologies Used

  • React.js: For building a dynamic and component-based user interface.
  • Chart.js / Recharts: For creating interactive and visually appealing data visualizations.
  • REST APIs: For fetching real-time data from various government sources.
  • CSS Modules / Styled-Components: For scoped and maintainable styling.

Project Overview

The goal of this project was to build a user-friendly platform that demystifies complex data related to India's ethanol production and electric vehicle (EV) adoption. By integrating with public government APIs, the dashboard fetches, processes, and displays this information through interactive charts and graphs, making it accessible and understandable for the general public, researchers, and policymakers.

Key Features

  • Real-Time Data Integration: Asynchronously fetches the latest data from multiple APIs to ensure the information is always current.
  • Interactive Data Visualization: Implemented dynamic bar charts, line graphs, and pie charts to allow users to explore data trends and comparisons intuitively.
  • Component-Based Architecture: Built with a modular structure in React, making the codebase clean, scalable, and easy to maintain.
  • State Management: Managed application state effectively, including loading and error states, to provide a smooth user experience while data is being fetched.

Challenges & Problem-Solving

The Challenge:
The biggest technical hurdle was handling the inconsistency and unreliability of the public APIs. Data from different sources arrived in varied formats (XML, JSON), with different key names, and sometimes the API endpoints would be slow or unresponsive. This could lead to a broken UI or a poor user experience.

The Solution:
To solve this, I designed a dedicated data normalization layer within the application. When data is fetched, it first passes through this layer before being stored in the application's state. This layer is a set of utility functions that:

  • Transforms all incoming data into a consistent JSON format.
  • Standardizes key names and data structures.
  • Implements robust error handling for failed API calls, displaying a user-friendly message instead of crashing the app.
  • Caches data temporarily to reduce the number of API calls and improve performance.
This approach decoupled the UI components from the messy data sources, making the front-end code much cleaner and more resilient to back-end issues.
